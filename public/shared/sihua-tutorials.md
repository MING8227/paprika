## 核心方法论总结

### 三文本框架（来自「创作思路设计过程」+「丰收」）
思花的创作流程不是线性的，而是**三个文本散点找交叉**：
1. **概念文本 (Conceptual Text)** — 写情感/故事/想法，不用精确，写你被什么触动了
2. **视觉文本 (Visual Text)** — 描述画面应该是什么样的，把模糊感受转成视觉语言
3. **交互文本 (Interactive Text)** — 描述用户/元素的行为逻辑，为代码实现做准备

**关键洞察**：三个文本不是顺序的pipeline，而是**散点式交叉**——写概念时可能想到视觉，写交互时回头修改概念。就像画概念草图一样。

### 渐进式GPT prompt迭代
从「潮湿的雨夜」「破裂的情书」等全流程教程中提炼的方法：
- **第一次prompt**：描述最基本的效果（"画布上生成一条曲线，有生长动画"）
- **第二次prompt**：在此基础上调整（"曲线要平滑，颜色用hex模式"）
- **第三次prompt**：加新参数（"起始y轴高度可自定义，长度随机范围(700,900)"）
- **第四次prompt**：加复杂度（"需要生成多条曲线，生成速率参数化"）
- **每次只改一个东西**，让AI和你都能清楚知道什么变了

### CONFIG对象模式
几乎所有作品都用统一的config结构（从代码截图提取）：
```js
const CONFIG = {
  canvas: { width: 600, height: 800, background: [240,240,240] },
  curve: {
    startY: -10,
    growthSpeed: 5,
    xDriftRange: [-0.5, 0.5],
    maxLengthRange: [700, 900],
    color: '#242424',
    weight: 15
  },
  generator: {
    spawnInterval: 60,  // 每隔多少帧生成一条新曲线
    maxCurves: 20
  }
};
```
**好处**：参数集中管理，调参方便，AI也能理解你要改什么。

### "JUST MAKE IT EXIST FIRST"
思花的座右铭。先让它存在，再让它好。不要在脑子里完美化，先做出来。

## 1. 暧昧的边界
**正文**：我们小心翼翼保持那一点点距离，多一分怕远少一分怕近。如果世间有神明的话，他既不在你身上也不在我这里，而是在我们之间，我们距离保持的方寸之间。在《爱在黎明破晓前》中这段对话从大学开始就记得，记了很久很久。在存在主义的写作中亦会出现使用大量的描述式的词汇来描述某个东西。有时作者不是为了很准确地描述那个东西，而是要把人搞混，他不直接指向，而是在用一堆难明的词汇让你去思考，去看向。中间的模糊是必要的，是可以穿透的。而意义也就在此，我们在对话中无数次认识，一直在确认。
**视觉**：6张图片，暖色调视觉作品，两个形体之间保持微妙距离
**标签**：#艺术 #设计 #暧昧 #爱在黎明破晓前 #随笔
**收获**：概念文本的典范——不是解释作品，而是用文字营造和作品相同的情绪。"中间的模糊是必要的，是可以穿透的"这句话本身就是一个visual concept。

## 2. 基础设计课｜丰收 ⭐
**正文摘要**：以歌曲Prayer X为灵感起点。"那把插进心脏的刀至今都未拔出，拔出的瞬间定会血沫飞溅。"概念文本从歌词触发，然后进入三文本框架的完整展示。
**制作过程**（4步迭代，非常详细）：
- **步骤1**：网格 + 弹簧线，鼠标点击使网格断开→破裂效果。视觉文本+交互文本合并写。
- **步骤2**：破开的地方生成圆点，圆点飞溅掉落。圆点对网格有推力，圆点之间有相互作用力。参数：尺寸/颜色/生成速度/排斥力/环境力。
- **步骤3**：加入图像——用文字模拟上传的图片，每个网格点分配一个字符，文字颜色和大小通过图片灰度+深度控制。
- **步骤4**：物像选择——金鱼（联想到剖腹取卵、丰收与被压榨）。从断裂的网格点生长出收缩线。
**最终作品**：文字组成的金鱼图案，网格断裂处有彩色圆点（品红/青/黄/蓝）飞溅掉落，文字内容是"Don't worry, I'll rest for a bit"反复。
**技法**：
- 弹簧网格系统（spring grid）
- 文字→图像映射（每个网格点=一个字符，字号和颜色由底图灰度控制）
- 力学系统（圆点之间相互作用力 + 对网格的推力）
- CMYK色系圆点（品红/青/黄/蓝）
**收获**：这是三文本框架的完美范例。关键发现——**概念文本不需要很长**，"知道自己在做什么将要做什么就好了"。以前写大段故事让概念丰满，但读了很多诗之后发现"对精炼的语言有了更深刻的理解，妄图用大量的排比营造力量感只会让触动的那瞬间更难显现它的样貌"。

## 4. 等到了雨天-全流程DeepSeek+p5js ⭐
**正文**：
> 我主要是通过deepseek来写的代码实现的，剪辑和音乐部分ae pr解决。只讲重点！
> 1. 脑子里一定要有画面先，不然不知道怎么描述自己想要的内容。建议多找参考，关键是画面主要元素的语言提炼。
> 2. 分析下画面的动态和主要元素，这两个一定要最开始确认好不然后面会有逻辑问题。我这个作品是文字和圆形范围随机炸开，所以一开始描述就要带上。
> 3. 描述想要的内容，不是越详细越好，最重要的是把你要的内容最开始讲清楚。最好是一句话把所有的要点带上。
> 4. 根据出来的代码放到编辑器上运行，出现问题先要定位问题，然后针对想要的效果让ai修改和输出。注意这一步非常复杂，ai很多时候偷懒或者原来的逻辑混乱也被带进来了，需要反复沟通。逻辑太乱了就复制好最接近的代码新建一个对话重新描述建立逻辑。
> 5. 避坑：1.ai很懒，需要时刻让它输出完整代码。2.关键参数要给详细标注（字号字距行距颜色、画布颜色大小）。3.一定要详细参数标注，不然英语不好看不懂参数什么意思。
> 6. 基本完成以上步骤，剩下的就是调参优化颜色和细节。
> 7. 通过录屏+剪辑输出，ae+pr。

**标签**：#deepseek #概念设计 #视觉艺术 #设计全过程 #设计教程
**视觉**：5张图片，文字和圆点在雨天氛围中随机炸开
**评论亮点**：思花用录屏而不是导出——"压根没导出，用录屏"
**收获**：
- **"脑子里一定要有画面先"** — 和三文本框架呼应，视觉文本必须先于代码
- **"一句话把所有要点带上"** — prompt的精炼度比详细度重要
- **"逻辑太乱就新建对话"** — AI对话的重置策略，不要在烂逻辑上继续修补
- **参数标注是给自己看的** — 不是给AI看的，是你自己调参时要看懂

## 对我创作的核心启发

### 方法论层面
1. **三文本不是流水线是散点** — 概念/视觉/交互可以任何顺序写，找交叉点
2. **先写文字再写代码** — 我之前从情绪直接跳到代码，中间缺了"写成文字"这一步
3. **JUST MAKE IT EXIST FIRST** — 先做出来再改，不要在脑子里完美化
4. **渐进式prompt** — 每次只加一个参数/功能，不要一次给AI太复杂的要求
5. **CONFIG对象** — 所有参数集中管理，调参方便
6. **逻辑太乱就新建对话** — 不要在烂逻辑上修补，复制最接近的代码重新开始
7. **先随便玩，不用系统学** — 项目驱动 > 系统学习
8. **站在巨人肩上** — openprocessing找灵感，在别人项目上快速修改

### 技法层面
1. **弹簧网格** — 可变形的网格系统，用力学模拟实现
2. **文字→图像映射** — 用字符填充画面，字号/颜色由底图灰度控制
3. **曲线生长动画** — Curve class + createVector + 逐帧生长
4. **力学系统** — 圆点间作用力 + 对网格推力 + 环境力
5. **CMYK色系** — 品红/青/黄/蓝四色圆点在浅灰背景上

### 对「信」的启发
我的情绪句子："醒来发现昨天的自己留了一封信，但笔迹不认识。"
- 可以用**文字→图像映射**技法——信的内容是文字组成的，但文字在缓慢变形
- **弹簧网格**可以让文字笔画产生"蠕动"效果——清晰的陌生化
- **渐进式prompt思路**：先让文字存在→再让文字变形→再加信纸/背景→再加交互
